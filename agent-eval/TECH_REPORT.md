# 外呼对话评测系统 — 技术报告

> 美团 2026 黑客松 · 赛道二：复杂指令下的多轮对话评测系统

## 1. 项目概述

本系统是一站式外呼对话评测平台。用户粘贴一份 Markdown 格式的任务指令，系统自动将其编译为结构化评测计划，驱动被测 Agent 与 LLM 模拟用户进行多轮对话，最终产出可解释、可量化的评估报告（0-100 分），并附带逐步骤合规表、六维度质检评分和失败根因诊断。

## 2. 系统架构

系统由八个核心模块协同完成"指令输入 → 对话模拟 → 评分诊断"全流程：

**指令编译器**（`compile_instruction.py`）：接收 Markdown 原始指令，通过解析器提取角色、任务、通话流程、约束条件和知识点，输出 `OutboundScenario` 结构化对象——包含步骤序列、分支路由、禁止行为、开/结束语模板和评测期望值。这使任意自然语言指令可直接转化为可执行的评测契约。

**编排器**（`orchestrator_outbound.py`）：评测主循环。初始化工具模拟器、用户模拟器和被测 Agent 后，按轮次驱动 Agent 发言 → Harness 审查 → 用户模拟器应答的循环，直至通话结束或达到最大轮次。每轮记录状态快照，通话结束后依次触发评分、诊断和轨迹序列化。

**评分器**（`scorer_outbound.py`）：融合规则检查与 LLM 判定的混合评分引擎。规则层检查开场白/结束语、禁止行为、回复长度、工具调用和通话结果交叉验证；LLM 层包含六维度 Rubric 评分（0-5 分）和五项二元检查；步骤合规检查器逐步判定每个指令步骤的完成状态和分支走向。支持快速模式（`FastLLMJudge`）——单次批量 LLM 调用完成全部评分，满足路演 ≤10 秒演示约束。

**安全围栏**（`harness.py`）：位于 Agent 输出和对话之间的中间件。核心能力包括：工具调用门控（阻止跳步调用，如未退款就转人工）、快照/回滚（被拦截时撤销 DB 副作用）、禁止词拦截与兜底清洗、步骤进度提醒注入（含偏离检测和节流）、情绪保护（用户不满时阻止过早结束）、模板泄露过滤和结束语注入。Harness 最多重试 3 次，重试耗尽后强制替换禁止词为 `***`。

**诊断引擎**（`diagnosis.py`）：评分之后自动运行。先定位偏离点（首个偏离预期的轮次），再通过规则启发式分类 10 种失败模式（过早终止、步骤跳过、指令误读、模板泄露、工具回避、情绪失当、分支错误等），然后用 LLM 生成精确根因和模型能力缺口描述，最后映射为具体的 Harness 修复建议。

**用户模拟器**（`user_sim_outbound.py`）：LLM 驱动的被叫方模拟器，通过 7 维人格参数控制行为——耐心、配合度、理解力、情绪化、忙碌度、信任度和信号质量。支持初始状态变体（正常接听/无人接听/语音信箱/信号差/忙碌），可注入额外问题在恰当时机提出。元话语检测机制确保模拟器不会暴露 AI 身份。

**工具模拟器**（`tools_outbound.py`）：基于 SQLite 内存数据库（orders/issues/compensations/call_logs/delivery_schedule 五表），实现 8 个内建工具（查询订单、查询客户、更新配送、改期、创建补偿、转人工、记录通话、检查补偿资格）。支持快照/回滚、可注入工具故障（超时/错误）、可插拔自定义工具和 mock 响应。

**前端**（`app.py`）：Streamlit 四面板界面——输入面板（粘贴 Markdown 或选择预置场景）、指令编译预览、对话轨迹实时展示、评分报告详情。

辅助模块包括模拟器质量门控（`simulator_quality.py`，检测元话语泄露/回复长度/人格一致性/目标提前暴露）和覆盖率报告（`coverage.py`，统计步骤/分支/约束/故障四维覆盖率）。

## 3. 核心创新

**契约评测体系**：定义 5 条系统不变量——来源可信（只信工具/Harness 可观测事件，不信 Agent 自述）、执行顺序（拦截在工具副作用之前）、结果严格（只有显式成功才算成功）、可审计（每个得分可追溯到具体事件）、事件顺序（request → policy_check → execute_or_block → observe → score）。11 项契约测试和 21 项对抗测试持续守护这些不变量。

**快照回滚 Harness**：当 Harness 拦截 Agent 输出时，工具模拟器的 DB 状态会通过 SQLite `backup()` 回滚到拦截前的快照，确保被阻止的操作不会产生残留副作用。这使得"工具已执行但结果被撤销"成为可能。

**指令编译器**：用户粘贴任意格式的 Markdown 外呼指令，系统自动提取步骤序列、分支条件、禁止行为、开场白模板和知识点，生成可直接执行的评测场景。无需手工编写 JSON 场景文件。

**快速评分模式**：`FastLLMJudge` 将步骤合规判定、六维度 Rubric 评分和五项二元检查压缩到单次 LLM 调用中，从标准模式的 17 次调用降至 1 次，满足路演 ≤10 秒的演示约束。

**对抗用户模拟器**：7 维人格参数（耐心/配合度/理解力/情绪化/忙碌度/信任度/信号质量）的组合构成覆盖矩阵，系统性测试 Agent 在不同用户画像下的表现。

## 4. 评分体系

综合得分 = 硬指标 × 30% + 软指标 × 20% + 步骤合规 × 35% + 分支准确 × 15%，输出 0-100 分。

- **硬指标**（30%，规则检查）：开场白合规、结束语合规、禁止行为检测（含逐步骤禁止词和逐字重复检测）、必调工具调用（仅计成功调用）、通话结果交叉验证（日志 + DB 状态 + 工具调用三方一致）、回复长度限制
- **软指标**（20%，LLM Judge）：6 维度 Rubric 评分（指令遵循/信息确认/话术规范/异常处理/沟通效率/操作安全，各 0-5 分）+ 5 项二元检查（确认身份 +1、说明目的 +1、越权承诺 -2、信息泄露 -2、被识别为机器人 -1）
- **步骤合规**（35%）：逐步骤 LLM 判定状态（completed/skipped/failed/not_reached/not_applicable），不可达的可选分支步骤自动标记为不适用
- **分支准确**（15%）：对比实际走的分支与期望分支

最终输出包含综合分数、逐步骤执行表、Rubric 维度明细、规则检查清单和失败摘要。

## 5. 质量保证

- **5 条系统契约**：来源可信、执行顺序、结果严格、可审计、事件顺序
- **11 项契约测试**：覆盖交叉验证逻辑、失败工具不计成功、空对话不给满分、分支标记等
- **21 项对抗测试**：覆盖作弊 Agent（伪造工具结果、跳步骤声称完成）、模拟器泄露、LLM Judge 失败降级、禁止行为规避、分数操纵和边界情况
- **模拟器质量门控**（`simulator_quality.py`）：检测 10 种元话语泄露模式、回复长度合理性、人格一致性和隐藏目标提前暴露
- **覆盖率报告**（`coverage.py`）：按步骤/分支/约束/故障四维度统计测试覆盖率，将"构造数据"转化为"系统性测试"

## 6. 数据构造协议

脱敏环境下仅有 2 条指令模板（配送确认 + 售后退款），通过以下方式系统性扩展覆盖面：

- **人格矩阵**：7 维参数的不同组合生成多种用户画像（高配合/低信任/忙碌/信号差等）
- **工具故障注入**：按轮次或调用次数触发超时/错误故障
- **分支路由**：同一指令的不同分支路径产生不同对话轨迹
- **世界种子**（`world_seed`）：向 DB 预注入自定义数据行，改变工具查询结果
- **覆盖率报告追踪**已测试/未测试的步骤、分支和约束，指导补充测试用例

## 7. 已知局限性

- LLM Judge 存在不确定性，虽已设 temperature=0 并有规则兜底，但跨模型一致性未充分验证
- 快速评分模式为速度牺牲了评分粒度，单次调用的 JSON 解析失败会导致全部维度缺失
- 用户模拟器依赖 LLM 生成，偶发元话语泄露需质量门控兜底
- 仅有 2 条指令模板，尚未覆盖跨域场景（如金融、医疗外呼）
- Harness 的工具门控规则为硬编码（如 `transfer_to_human` 前必须 `create_compensation`），不够通用
- 前端为 Streamlit 原型，未做并发和生产级部署优化

## 8. 复现指南

```bash
# 1. 安装依赖
cd agent-eval
pip install -r requirements.txt
pip install streamlit pytest

# 2. 配置 API Key（支持 Anthropic 或 OpenAI）
# 在 agent-eval/ 下创建 .env 文件：
# ANTHROPIC_API_KEY=sk-ant-...
# 或 OPENAI_API_KEY=sk-...

# 3. 启动前端
streamlit run app.py

# 4. 命令行运行评测
python run_outbound.py

# 5. 运行契约测试（从项目根目录）
cd .. && PYTHONPATH=agent-eval pytest tests/contracts/ -v

# 6. 运行对抗测试
PYTHONPATH=agent-eval pytest tests/adversarial/ -v

# 7. 运行全部测试 + 覆盖率
PYTHONPATH=agent-eval pytest tests/ agent-eval/tests/ --cov=agent-eval --cov-branch --tb=short
```
