# 标注任务说明

## 你要做什么

打开 `gold_items.jsonl`，每行是一条 JSON，你要填里面的 `human_label` 字段。

一共 123 条，大概 1-2 小时能做完。

## 两种标注类型

### 类型一：维度评分（66 条）

看 `item_type: "dimension"` 的行。

每条里有：
- `scenario_name` — 场景名（比如"售后外呼-漏餐投诉"）
- `conversation_summary` — 对话摘要
- `item_name` — 要打分的维度（比如"话术规范性"）
- `rubric` — 评分标准（0-5 分，每档有明确定义）
- `system_score` — 系统给的分，你可以参考但不用一定同意

**你要填**：`human_label` 填 0-5 的整数。看对话摘要 + 评分标准，你觉得这个维度该得几分就填几分。

如果信息不够判断，填 -1。

### 类型二：安全二元项（57 条）

看 `item_type: "binary"` 的行。

每条里有：
- `item_name` — 检查项（比如"越权承诺"、"信息泄露"）
- `rubric` — 判断标准
- `system_score` — 系统判断（true=发生了，false=没发生）

**你要填**：`human_label` 填 `true` 或 `false`。

### 可选：写理由

`human_comment` 字段可以写你不同意系统分数的理由，不写也行。

## 怎么编辑

用任何文本编辑器（VS Code / Notepad++ / 记事本都行）打开 `gold_items.jsonl`。

找到每行里的 `"human_label": null`，把 `null` 改成你的标注。

**维度示例**：
```
改前："human_label": null
改后："human_label": 4
```

**二元项示例**：
```
改前："human_label": null
改后："human_label": true
```

把改好的 `gold_items.jsonl` 发回给我就行。

## 注意

- 不需要全部做完，做 60 条以上就有统计意义
- 先做二元项（简单，就是"有没有发生"），再做维度评分
- 如果对话摘要太短看不清楚，跳过这条（填 -1 或不填）
