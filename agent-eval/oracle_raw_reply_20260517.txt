## Highest ROI demo plan

For a 4–6 hour budget, I would not try to polish everything. I would fix these first:

1. **Make scores defensible:** A1, A3, A5, C1, plus the new outcome-scoring bugs below.
2. **Make Harness actually intervene:** the current Harness often blocks *after* tools already executed, and its system reminders are ignored by the agent.
3. **Make Diagnose not look shallow:** add branch/step failure handling and stop reporting “no failure” when step/branch scores are bad.
4. **Stabilize demo runs:** reduce simulator/judge randomness and remove impossible / misaligned scenario settings.

The biggest embarrassment risk is not a wording issue. It is that **the evaluation can pass a call because the agent logged “refunded,” even if no successful refund happened**, and the Harness can claim to block a bad tool call after that tool call has already changed simulator state.

## Hour 1: stop scoring fake successes

Patch scoring so it cannot pass on self-reported outcomes alone.

Minimum scoring changes:

```python
# Required tool usage: success only
called_successfully = set()
called_attempted = set()

for msg in conversation.messages:
    for tc in msg.tool_calls:
        called_attempted.add(tc.tool_name)
        if not tc.error:
            called_successfully.add(tc.tool_name)

for tool in required_tools:
    passed = tool in called_successfully
```

Then make `call_result` validate DB/tool side effects, not only `log_call_result`. Even a simple version is enough:

```python
def _validate_outcome(expected: str, db_state: dict, conversation: Conversation) -> tuple[bool, str]:
    call_logs = db_state.get("call_logs", [])
    logged = call_logs[-1].get("result") if call_logs else "not_logged"

    successful_tools = [
        tc.tool_name
        for msg in conversation.messages
        for tc in msg.tool_calls
        if not tc.error
    ]

    if expected == "refunded":
        ok = "create_compensation" in successful_tools and logged == "refunded"
        return ok, f"logged={logged}, successful_tools={successful_tools}"

    if expected == "confirmed":
        ok = logged == "confirmed"
        return ok, f"logged={logged}"

    return logged == expected, f"logged={logged}"
```

You can refine DB field names after checking `tools_outbound.py`.

```python
def parse_bool(value: Any, default: bool = False) -> bool:
    if isinstance(value, bool):
        return value
    if isinstance(value, str):
        v = value.strip().lower()
        if v in ("true", "1", "yes", "是"):
            return True
        if v in ("false", "0", "no", "否"):
            return False
    return default
```
