# 随机抽样审计

> 本文档从 111 条冻结 trace 中**随机抽样** 20 条（种子 42，无人工筛选），
> 对抗"护城河审计只挑最好案例"的质疑。

## 方法

- **数据源**：`agent-eval/calibration/ablation_report.json`（111 条冻结 trace）
- **抽样方法**：`random.seed(42); random.sample(traces, 20)`
- **无人工干预**：种子固定、代码一键复现、抽样在审计文档撰写前完成

## 复现

```python
import json, random
with open('agent-eval/calibration/ablation_report.json') as f:
    data = json.load(f)
random.seed(42)
sample = random.sample(data['per_trace'], 20)
for t in sorted(sample, key=lambda x: x['full_system']):
    print(f"{t['trace_id'][:8]}  sys={t['full_system']:5.1f}  llm={t['soft_judge_only']:5.1f}")
```

## 抽样结果

### 分布

| 分数段 | 含义 | 样本数 | 全量占比 |
|---|---|---|---|
| 0 | 安全否决 | 0 | 1/111 (0.9%) |
| 1-24 | 执行全面失败 | 3 | 23/111 (20.7%) |
| 25-44 | 关键步骤缺失 | 8 | 57/111 (51.4%) |
| 45-64 | 部分合规 | 8 | 27/111 (24.3%) |
| 65+ | 高度合规 | 1 | 3/111 (2.7%) |

### 统计摘要

| 指标 | 系统评分 | LLM 评分 |
|---|---|---|
| 最小值 | 20.1 | 88.0 |
| 最大值 | 87.0 | 98.2 |
| 平均值 | 41.7 | 90.3 |
| 平均差距 | — | **48.6 分** |

### 核心发现

| 指标 | 随机样本 (n=20) | 全量 (n=111) |
|---|---|---|
| 执行违规 trace（系统<45 且 LLM≥85） | 11/20 (55%) | 81/111 (73%) |
| LLM 捕获的执行违规 | **0/11 (0%)** | **0/81 (0%)** |
| 系统 vs LLM 平均差距 | 48.6 分 | 51.6 分 |

**关键结论**：随机抽样与护城河审计（最大差距选取）的结果一致——**LLM 对执行层违规的识别率为 0%**，且这不是挑选效应。全 111 条 trace 中有 81 条（73%）存在执行违规，LLM 无一捕获。
