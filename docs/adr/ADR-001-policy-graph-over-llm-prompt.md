# ADR-001: 用策略图替代 LLM Prompt 做评分基准

**状态**: 已采纳
**日期**: 2026-04-15
**决策者**: 团队

## 背景

评测 Agent 多轮对话质量有两条路：
1. **LLM-as-Judge**：把对话和评分标准塞进 prompt，让 LLM 打分
2. **结构化验证**：把业务策略编译成可执行的数据结构，用确定性规则验证

## 决策

选择路线 2：将业务指令编译成 PolicyGraph（有向图，节点=步骤，边=条件分支，约束=时序），评分基于对话轨迹与图的对齐程度。

## 理由

- **LLM 独评均分 88.8%**：消融实验证明 LLM 评委好坏不分，所有 Agent 都是"好学生"
- **可审计性**：每个失分点指向具体的图节点和事件证据，评委可以一键验证
- **确定性**：同一 trace 跑两次得分一致，中位数标准差 7.1%（仅来自 12% 的 LLM 软质量层）
- **条件分支**：SOPBench 作者自述不支持条件分支（论文 Limitation 第 2 条），我们原生支持

## 代价

- 需要为每个场景手写 PolicyGraph JSON（当前 34 个场景），编写成本高于 prompt
- 新业务场景需要人工将策略文档翻译成图结构
- 图覆盖率受限于场景作者的经验

## 替代方案

- LLM-as-Judge（rejected）：均分 88.8%，不可接受
- LLM + rubric（rejected）：比纯 LLM 好，但仍不可审计、不确定
- 纯规则匹配（rejected）：不支持多条件分支和时序约束

## 验证

- 消融实验：完整系统 37.2% vs LLM 独评 88.8%，差 51.6pp
- 去掉步骤合规组件后均分 +8.1pp，证明策略图对齐是最大贡献者
