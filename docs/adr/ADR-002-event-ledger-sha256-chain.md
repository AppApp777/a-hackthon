# ADR-002: EventLedger + SHA-256 哈希链做不可变审计日志

**状态**: 已采纳
**日期**: 2026-04-20
**决策者**: 团队

## 背景

Agent 可能伪造工具调用结果（声称"已退款"但实际未调用退款工具）。评测系统需要一种防篡改机制来区分真实执行和自报声明。

## 决策

引入 EventLedger：每个工具调用、状态变更、harness 事件都写入 append-only 日志，每条事件包含前一条的 SHA-256 哈希，形成不可变链。评分前自动调用 `verify_chain()`，链断裂则拒绝评分。

## 理由

- **来源可信契约**（CONTRACTS.md #1）：打分只看可观测事件，不看 Agent 自报
- **防篡改**：中间事件被删除、插入、交换都会导致哈希链断裂
- **学术支撑**：ESAA（arxiv 2602.23193）独立提出了相同的 Event Sourcing + SHA-256 投影哈希方案，验证了设计方向
- **离线校验**：`chain_hash()` 返回整条链指纹，写入 trace 文件，支持评委离线验证

## 代价

- 每条事件多一次 SHA-256 计算（微秒级，可忽略）
- trace 文件体积增大约 15%（每条事件多一个 hash 字段）
- 链式结构要求严格顺序写入，不支持并行追加

## 验证

- `tests/contracts/test_hash_chain.py`：16 项测试覆盖篡改、删除、插入、交换、空链等场景
- 对抗测试 `tests/adversarial/test_anti_gaming.py`：验证伪造执行记录被检测
