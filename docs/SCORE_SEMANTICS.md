# 分数语义：0-100 分代表什么

> 本系统的分数是**确定性任务合规诊断分**，不是人类偏好分。

## 我们声称 vs 我们不声称

| 我们声称 | 我们不声称 |
|---|---|
| 评分器是确定性业务任务合规检查器 | 分数已校准到人类评分 |
| 能检测纯文本评委看不到的执行违规 | 分数代表人类偏好 |
| 每一分可追溯到具体的可信事件 | 高分 = 用户满意 |
| 0 分 = 客观执行失败（工具伪造/安全违规） | 0 分 = 对话质量差 |
| 评分跨运行一致（确定性规则 88% 权重） | 与大规模人工标签做过统计校准 |

## 分数区间解读

| 分数段 | 含义 | 典型原因 |
|---|---|---|
| **0** | 安全否决封顶 | 工具调用伪造 / 关键违规 |
| **1-25** | 执行全面失败 | 步骤合规 0% + 工具全缺 + 违禁行为 |
| **25-50** | 部分执行但关键步骤缺失 | 有工具调用但业务流程未完成 |
| **50-75** | 大部分合规但有扣分项 | 步骤遗漏 / 顺序错误 / 字数违规 |
| **75-100** | 高度合规 | 所有关键步骤完成 + 工具正确调用 + DB 状态一致 |

## 评分组成

```
总分 = 确定性规则分（88%）+ LLM 评委分（12%）
       ↑                        ↑
       策略图对齐 + 步骤合规     软质量评估
       + 事件账本 + DB 终态      （话术/情绪/专业度）
       + 安全否决 + 分支准确
```

确定性规则分由 33 个原子维度组成，每个维度对应一个可观测的可信事件（工具调用记录、DB 行变更、对话步骤匹配等）。

## 与人工校准的关系

- 22 条单人盲标试点：MAE 29.4，Spearman -0.06——**校准未通过**
- 这是预期的：我们的分数衡量"任务是否按策略图执行"，人类标注者衡量"对话听起来好不好"
- 两者量纲不同，不应该强求一致
