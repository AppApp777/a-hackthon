核心亮点

不是"让 LLM 判 LLM"

⚙

真工具执行 核心思路

Agent 调用的每个工具（查询订单、补偿操作、用户信息查询）都有真实模拟执行， SQLite 内存数据库记录状态变更，不依赖 Agent 自报结果。

18 工具
🔒

SHA-256 哈希链

每个评测事件（工具调用、评分判定、干预记录）都上链哈希， 形成篡改可检测的审计轨迹（任何事件被插入、删除、修改、交换均会被 verify_chain() 捕获）。

✅

三层确定性评分

客观证据层（88%）完全由规则计算， 软质量层（12%）由 PoLL 双模型评委交叉验证并被客观分门控， 安全否决层伪造直接封顶 0 分。

88%
📈

因果诊断

不只打分——定位最小失败点 + 估算修复后恢复多少分。 "第 5 步走错分支，加资格核查，+18 分"。 评测变成工程改进循环。

🛡

反作弊三层防御

Agent 说"我查了订单"但账本没有 query_order 记录 → 判伪造，封顶 0 分。 表面防御 + 轨迹防御 + 基准防御。

📊

校准验证

二元判定 Cohen's κ = 0.868（近乎完美一致性）。 GPT-5.5-pro 32 条独立交叉验证，67% ±1 维度一致。 声称的数字全部可复现。
