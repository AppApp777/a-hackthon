**算分**：
- 创新性（25 分）：工具执行全场唯一，预计最高分
- 完整性（25 分）：无演示、无 Docker、缺知识准确性，可能丢 8-12 分
- 应用效果（25 分）：无图表、无在线演示，可能丢 5-10 分
- 商业价值（25 分）：与头部竞品持平

**接下来 12 小时必须做**：
1. GitHub Pages 演示（3h）—— 非此即彼，这是"评委看不到"和"评委能点开"的分界线
2. Dockerfile（1.5h）—— 非此即彼，专家会尝试跑代码
3. 仪表盘雷达图 + 热力图（3h）—— 高优，视觉冲击力直接影响"应用效果"得分

**做了这三件**（7.5 小时）：从"最强但看不见的引擎"变成"最强且专业呈现的引擎"，排名 1-2。

**不做这三件**：排名 3-5。技术深度被那些看起来完整、容易试用的竞品淹没。

### 有人补齐我们的差距吗？——头条：核心护城河无人触及

12 个竞品**没有一家**有：真实工具执行（18 工具 + SQLite 状态变更 + 事务回滚）、SHA-256 哈希链防篡改账本、子进程沙箱、因果失败链诊断 + 反事实修复、消融实验（LLM 88.8 vs 全栈 37.2）。所有竞品本质都是"模拟用户对话 → LLM 给文本打分"，**没有"Agent 真去调工具"这个概念**——所以伪工具调用检测、工具执行打分对他们不是"缺失"，而是架构上压根不存在。

发生的"软差距"收窄（都在校准/测试/前端周边轴，非护城河）：

- **evalcall（最值得关注）**：① 加 43 条确定性单测，**抹掉了"零测试"标签**（但是常规单测，不是变质测试）；② 砸重投校准严谨度——盲标 held-out 验证集（标注者与裁判调优链隔离，正面反驳"同源过拟合"）、按模型分别置信度校准、预注册机制（出分前落盘，实测拦掉一次 90.3→73.3 坏迭代），**这是"评委审严谨性"轴上的真功夫，比我方记录的强**；③ 但它自称的"消融"是单任务约束开/关（均值 68.2），**和我方"裸 LLM 88.8 vs 全栈 37.2"是两码事、更弱的主张**；④ 它的"跨模型"只是 claude CLI 的 haiku/sonnet/opus，**不是我方 7 个异构真实模型**。
- **CGADS**：报告挂 `evidence_ref`（对话轮级，非原始 span 级）、修了"0%覆盖却 PASS"的 bug、加 4 个对抗画像。**全在对话模拟框架内。注意：13 个功能提交后测试数仍是 6 个，零新增测试**——我方"测试薄/无执行验证"护城河对它原样成立，甚至更宽。
- **EvalSystem**：只补了"前端一般"（5 页仪表盘 + Vercel SPA + Docker 一键部署），**零评测能力增量**；它 README 里直指我方护城河的路线图项（多 Judge 集成、校准集扩 50+）仍是未勾选 TODO。

- **evalcall 把"严谨性叙事"做扎实了**（盲标 held-out 94.7% + 预注册防 p-hacking + 按模型校准 + 跨模型裁判团），正好打在美团赛道二最吃重的"评委审代码 + 方法论严谨"那条轴上。我方人工校准/盲测叙事若不打磨，**现场答辩可能被它的"诚实化大修/预注册拦坏迭代"叙事盖过气场**——不是技术输，是"显得更严谨"。
