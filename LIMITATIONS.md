# 已知限制

> 诚实列出系统当前的局限性。评测系统的可信度不来自"没有缺点"，而来自"知道自己的缺点"。

## 评分层面

### 3. LLM 评委稳定性有限

消融实验显示 LLM 评委仅贡献 12% 权重、对总分影响 -0.3pp，这是刻意设计（不依赖不可靠信号）。但软质量维度（共情、清晰度）的评估精度仍受限于 LLM 能力。

### 3b. 步骤完成检测：确定性别名匹配的精度边界（2026-06-06）

**背景**：原实现把中文步骤指令整句当关键词，句内无空格导致永远匹配不上 → 纯对话步骤对所有模型恒判未完成（详见 BUGS.md「70 分天花板」）。已改为"语义动作→自然话术别名"概念词表（确定性、无 LLM、可复现），并加覆盖率门（多动作步骤需命中 ≥50% 动作）+ 按动作否定守卫。

**仍存在的边界**：
- **长尾未映射动作免检**：`_ACTION_CONCEPTS` 覆盖高频对话动作（self_identify/apologize/offer_refund 等），但 instruction_steps 里有一批一次性动作（如 estimate_duration、ask_packaging）未入表。一个步骤只要有任一动作映射成功，未映射动作就被静默跳过、不计入覆盖率分母 → 这些动作"免检"。影响集中在少数稀有场景，留出集主力场景已覆盖。
- **对话步靠 Agent 自报话术，交叉验证弱**：工具步靠可信事件（DB/ledger），对话步只有否定守卫一层防护。带强制工具结果的场景由 `result_correct` veto（cap 0.60）兜底，但纯对话场景（满意度回访/通知类）作弊收益相对高。
- **升级方向**：用确定性别名作规则兜底，叠加 LLM 语义步骤判定（系统已有 `SemanticUtterancePredicate`/`StepComplianceChecker` 骨架，温度 0）作主判，可进一步收窄精度边界。

## 数据层面

### 5. 校准数据：二元判定强，维度评分待完善

**现状**：二元判定（通过/失败）已有人工标注校准，κ=0.868（近乎完美一致性）。维度评分（D1-D6，0-5 分）已从 `round(hard_score * 5)` 粗暴映射重构为独立维度路由（`_compute_rule_dimensions()`，15+ 规则检查路由到 D1-D6），并用 GPT-5.5-pro 完成 32 条 trace 全量交叉验证（整体 67% ±1 一致，D1 步骤遵循 78% ±1 一致）。

**差距**：维度级校准虽有改善，但 Oracle 交叉验证的加权 κ 仍接近零，说明维度级评分与 Oracle 的一致性有限。±1 容忍度下一致性 67% 尚可，但精确一致性偏低。早期二元标注 κ=0.868 的标注数据未保留，当前不可独立复现。部分维度（D2 信息确认、D5 沟通效率）的规则信号较粗，需混合 baseline 补偿。

### 6. 元评测区分度：good vs medium 排序有限

**现状**：元评测三级排序中，medium ≥ flawed 排序 100% 正确（11/11 场景）。但 good（harness 辅助）≥ medium（裸跑）排序仅在 3/6 场景成立（50%）。在部分场景中，harness 的约束反而降低了 Agent 得分。

**原因**：harness 在安全护栏模式下会拦截或注入步骤，导致某些场景下 Agent 行为受限、反而丢分。这说明 harness 的"安全性"和"得分"之间存在取舍。

**影响**：系统能可靠区分好 Agent 和故意差 Agent（flawed），但区分"中等"和"好"的精度有限。这符合系统的设计目标——核心价值是捕获违规和伪造，而非精细排名。

### 7. 模型覆盖：3 个模型，未覆盖全部主流

**现状**：已测试 Claude Sonnet 4、MiMo-V2.5-Pro、LongCat-2.0-Preview 三个模型，20+ 条干净 trace。但未覆盖通义千问、文心一言、豆包等主流中文大模型。

**影响**：评分系统的泛化性在更多模型上未验证。部分 API key 已失效（Kimi 401、百炼 401），需用户重新获取。
